Editing Static Text on Pages

Most of the text on the site — page titles, descriptions, service times, the "About" paragraphs, ministry page copy — lives directly in public/index.html. This file contains the entire site.

Easiest way: Describe the change to Claude. E.g. "Change the Food Pantry page headline from 'Caring for Neighbors' to 'Feeding Our Neighbors'". Claude will find the exact spot and make the edit.

If you want to find it yourself

Open public/index.html and search (Ctrl+F / Cmd+F) for a unique phrase from the page you're editing — for example a few words from the heading or paragraph you want to change. Each page section starts with a comment like:

<!-- FOOD PANTRY -->
<div id="page-foodpantry" class="page">

Page IDs for each section of the site:

Search for this What page it is
id="page-home"Home page
id="page-worship"Worship page
id="page-about"About page (vision, mission, staff)
id="page-ministries"Ministries hub (all ministry cards)
id="page-music"Music Ministry
id="page-stephen"Stephen Ministry
id="page-foodpantry"Food Pantry
id="page-bees"Urban Beekeepers
id="page-christmasmarket"Christmas Market
id="page-youthfamily"Youth & Family hub page
id="page-youth"Youth Ministry (grades 7–12)
id="page-contact"Contact page
id="page-give"Give page

What you can safely edit vs. what to avoid

  • Safe to edit: Any visible text between > and < tags — headings, paragraphs, labels.
  • Safe to edit: URLs in href="..." values — links, email addresses.
  • Be careful with: Any line that has style="...", class="...", or JavaScript (onclick="...") — changing these can break layouts or navigation.
  • Don't touch: Lines that start with <script>, function, or contain fetch( — that's the code that makes the page work.

Note on admin-managed sections: Some parts of ministry pages show content from the admin portal (volunteer posts, seasonal blocks, ministry announcements). These are separate from the static HTML text. The static text is the "always-there" foundation; admin-managed content layers on top.

Adding a Header Photo to a Ministry Page

Each ministry landing page has a hero section at the top — the big colored banner with the page title. Right now they all show a solid steel-blue color. When you have a photo for a page, here's how to add it.

Pages that support header photos: Music, Stephen Ministry, Food Pantry, Urban Beekeepers, Christmas Market. The Christmas Market page already has its photo set.

Step 1 — Get the photo ready

Photos work best when they are:

  • Landscape orientation (wider than tall)
  • At least 1400px wide (wider = looks better on large screens)
  • JPG or WebP format (smaller file sizes)
  • Not too "busy" in the center — the text overlays the middle, so a photo with open sky or soft background works well

Step 2 — Upload the photo

Place the photo in /public/images/ministries/ in the website repository. Name it clearly, e.g. music-hero.jpg, stephen-hero.jpg, foodpantry-hero.jpg, bees-hero.jpg.

If you're working with Claude, just say: "Add this photo as the header image for the Music Ministry page" and share the file.

Step 3 — Edit the hero div in public/index.html

Find the ministry page section (search for e.g. id="page-music") and look for the hero div. It will have a comment above it saying exactly what to add:

<!-- Hero: add style="background-image:url('...')" when photo is available -->
<div class="page-hero page-hero--photo">

Change it to:

<div class="page-hero page-hero--photo"
     style="background-image:url('/images/ministries/music-hero.jpg');">

The overlay (dark blue tint that makes text readable) is applied automatically — you don't need to add anything else.

Easier option: Just tell Claude: "Add music-hero.jpg as the header photo on the Music Ministry page." Claude knows exactly where to put it.

How the Christmas Market photo is set (for reference)

The Christmas Market page already has its header photo. Here's what it looks like in the code — the same pattern applies to any ministry page:

<div class="page-hero page-hero--photo"
     style="background-image:url('/images/ministries/Christmas-MKT.jpg');">
  ...page title and subtitle here...
</div>

Editing Buttons on Ministry Pages

Every ministry landing page has a dark blue button bar at the very bottom of the page. Each button can be changed independently — you can edit the text, change the link, or hide a button you don't need.

Where to find them in the code

Each button bar is marked with a comment block that looks like this:

<!-- ══════════════════════════════════════════
     CHRISTMAS MARKET PAGE — CTA BUTTON BAR
     UPDATE ANNUALLY — change these each fall...
     ══════════════════════════════════════════ -->

Just search for the page name (e.g. CHRISTMAS MARKET PAGE — CTA BUTTON BAR) and you'll jump right to it.

Changing button text

The text that appears on the button is between > and </a>. Edit only that text:

<!-- Button 1: Vendor application -->
<a href="mailto:office@timothystl.org" class="btn btn-primary">Apply to be a vendor →</a>

Changing where a button links

Edit the href="..." value. Common patterns:

<!-- Link to a website -->
href="https://docs.google.com/forms/your-form-id"

<!-- Send an email -->
href="mailto:office@timothystl.org"

<!-- Pre-fill email subject line -->
href="mailto:office@timothystl.org?subject=Vendor Application"

Hiding a button temporarily

Add style="display:none;" to the <a> tag. Remove it to show it again:

<a href="..." class="btn btn-primary" style="display:none;">Apply to be a vendor →</a>

Easier option: Tell Claude what you want. For example: "Change the vendor button on the Christmas Market page to link to this Google Form: [paste URL]" — Claude will make the edit correctly.

Christmas Market — Annual Update Checklist

Do this each fall when the market details are confirmed, before registration opens.

Step 1

Update the date and time in the info box on the /christmasmarket page. Find the id="xmas-details" block and change the date text.

Step 2

Update the vendor application button with the new Google Form URL. Find the CHRISTMAS MARKET PAGE — CTA BUTTON BAR comment and change the href on Button 1 from the email address to the Google Form link.

It should go from: href="mailto:office@..."
To: href="https://docs.google.com/forms/d/YOUR_FORM_ID/viewform"

Step 3

Update the year in the info box. Find 2025 Market and change it to the current year.

Step 4 (optional)

Replace the header photo with a new one from this year's market. Upload the new photo to /public/images/ministries/, then update the background-image URL in the hero div. (See Adding a Header Photo above.)

After the market is over: Change Button 1 back to the email address so it doesn't link to an expired form. You can do this right away or wait until next fall.

Seasonal Worship Blocks

Seasonal worship blocks are optional content sections that appear on the Worship page (/worship). There are five, one for each major season. Each can be written once and then toggled on or off — so you can set up the Advent block in November, turn it on, then turn it off in January without losing your content.

Where to find them

Go to admin.timothystl.org/pages and look for:

  • Lent / Midweek worship — Midweek Lenten services, extra services during Lent
  • Holy Week & Easter — Palm Sunday, Maundy Thursday, Good Friday, Easter services
  • Thanksgiving worship — Thanksgiving Eve or Day service
  • Advent worship — Midweek Advent services, special series
  • Christmas services — Christmas Eve and Christmas Day service times

How to use them

Step 1 — Write the content

Click Edit on the seasonal block. Enter the service times, dates, and any description. You can use the rich text editor to add headings, bold text, or a simple table of service times. Click Save & Publish and check the "Show this block on the website" box.

Step 2 — Turn it on

Make sure the "Show this block on the website" checkbox is checked when you save. The block will appear on the Worship page immediately.

Step 3 — Turn it off when the season ends

Go back to the block, uncheck "Show this block on the website," and save. The content stays saved — ready for next year. The block disappears from the site.

Where does the block appear on the Worship page? It shows up between the "Service Times" section and the "Where Tradition Meets the Unexpected" section — right in the middle of the page where visitors will see it easily.

Community Concerts

The Music Ministry page has a dedicated Community Concert announcement section that appears when a concert is coming up. It is hidden between concerts and shown when one is scheduled.

Where to find it

Go to admin.timothystl.org/pages and click Edit on Community Concert announcement.

What to put in the content

Write the concert details in the rich text editor. Include:

  • The performer's name and a brief description
  • Date and time
  • Location (if different from the sanctuary)
  • Admission info (free? donations appreciated?)
  • Any other details

You can add a heading, bold text, and links (e.g. to the performer's website) using the editor toolbar.

Turning it on and off

Check "Show this block on the website" when you save to make it visible on the Music Ministry page. Uncheck it (and save) when the concert is over — your content stays saved for reference.

Updating for each new concert: Just go back to admin → Pages → Community Concert announcement, replace the content with the new performer's info, make sure the checkbox is checked, and save. No code changes needed.
