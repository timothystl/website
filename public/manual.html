Adding a Header Photo to a Ministry Page

Each ministry landing page has a hero section at the top — the big colored banner with the page title. Right now they all show a solid steel-blue color. When you have a photo for a page, here's how to add it.

Pages that support header photos: Music, Stephen Ministry, Food Pantry, Urban Beekeepers, Christmas Market. The Christmas Market page already has its photo set.

Step 1 — Get the photo ready

Photos work best when they are:

  • Landscape orientation (wider than tall)
  • At least 1400px wide (wider = looks better on large screens)
  • JPG or WebP format (smaller file sizes)
  • Not too "busy" in the center — the text overlays the middle, so a photo with open sky or soft background works well

Step 2 — Upload the photo

Place the photo in /public/images/ministries/ in the website repository. Name it clearly, e.g. music-hero.jpg, stephen-hero.jpg, foodpantry-hero.jpg, bees-hero.jpg.

If you're working with Claude, just say: "Add this photo as the header image for the Music Ministry page" and share the file.

Step 3 — Edit the hero div in public/index.html

Find the ministry page section (search for e.g. id="page-music") and look for the hero div. It will have a comment above it saying exactly what to add:

<!-- Hero: add style="background-image:url('...')" when photo is available -->
<div class="page-hero page-hero--photo">

Change it to:

<div class="page-hero page-hero--photo"
     style="background-image:url('/images/ministries/music-hero.jpg');">

The overlay (dark blue tint that makes text readable) is applied automatically — you don't need to add anything else.

Easier option: Just tell Claude: "Add music-hero.jpg as the header photo on the Music Ministry page." Claude knows exactly where to put it.

How the Christmas Market photo is set (for reference)

The Christmas Market page already has its header photo. Here's what it looks like in the code — the same pattern applies to any ministry page:

<div class="page-hero page-hero--photo"
     style="background-image:url('/images/ministries/Christmas-MKT.jpg');">
  ...page title and subtitle here...
</div>

Editing Buttons on Ministry Pages

Every ministry landing page has a dark blue button bar at the very bottom of the page. Each button can be changed independently — you can edit the text, change the link, or hide a button you don't need.

Where to find them in the code

Each button bar is marked with a comment block that looks like this:

<!-- ══════════════════════════════════════════
     CHRISTMAS MARKET PAGE — CTA BUTTON BAR
     UPDATE ANNUALLY — change these each fall...
     ══════════════════════════════════════════ -->

Just search for the page name (e.g. CHRISTMAS MARKET PAGE — CTA BUTTON BAR) and you'll jump right to it.

Changing button text

The text that appears on the button is between > and </a>. Edit only that text:

<!-- Button 1: Vendor application -->
<a href="mailto:office@timothystl.org" class="btn btn-primary">Apply to be a vendor →</a>

Changing where a button links

Edit the href="..." value. Common patterns:

<!-- Link to a website -->
href="https://docs.google.com/forms/your-form-id"

<!-- Send an email -->
href="mailto:office@timothystl.org"

<!-- Pre-fill email subject line -->
href="mailto:office@timothystl.org?subject=Vendor Application"

Hiding a button temporarily

Add style="display:none;" to the <a> tag. Remove it to show it again:

<a href="..." class="btn btn-primary" style="display:none;">Apply to be a vendor →</a>

Easier option: Tell Claude what you want. For example: "Change the vendor button on the Christmas Market page to link to this Google Form: [paste URL]" — Claude will make the edit correctly.

Christmas Market — Annual Update Checklist

Do this each fall when the market details are confirmed, before registration opens.

Step 1

Update the date and time in the info box on the /christmasmarket page. Find the id="xmas-details" block and change the date text.

Step 2

Update the vendor application button with the new Google Form URL. Find the CHRISTMAS MARKET PAGE — CTA BUTTON BAR comment and change the href on Button 1 from the email address to the Google Form link.

It should go from: href="mailto:office@..."
To: href="https://docs.google.com/forms/d/YOUR_FORM_ID/viewform"

Step 3

Update the year in the info box. Find 2025 Market and change it to the current year.

Step 4 (optional)

Replace the header photo with a new one from this year's market. Upload the new photo to /public/images/ministries/, then update the background-image URL in the hero div. (See Adding a Header Photo above.)

After the market is over: Change Button 1 back to the email address so it doesn't link to an expired form. You can do this right away or wait until next fall.

Volunteer Site

The volunteer signup site is at volunteer.timothystl.org. Admin access is at volunteer.timothystl.org/admin (same password).

Ministry pages with volunteer opportunities link directly to this site. When you add new volunteer events in the admin, they automatically appear in the linked button on those pages.

Asking Claude to Make Changes

Claude (the AI assistant) is the primary way changes get made to this site. You don't need to know how to write code — you just need to describe what you want clearly.

Good examples of requests

  • "Add this photo as the header image on the Food Pantry page: [attach file]"
  • "Change the vendor button on the Christmas Market page to link to this Google Form: [paste URL]"
  • "Update the Christmas Market page — the date is December 6, 2025, 10am–4pm"
  • "Add a button to the Bees page that links to [URL]"
  • "Change the heading on the Stephen Ministry page from 'You Don't Have to Face This Alone' to [new text]"
  • "Hide the vendor application button on the Christmas Market page until fall"

Tip: When asking Claude to update something on a specific page, always name the page (e.g. "Christmas Market page", "Food Pantry page"). This helps Claude find the right section quickly.
